## ⬡ &nbsp;About

<br/>

I am a **Full Stack Engineer** and **AI/ML Developer** based in Pune, India, currently pursuing **B.Tech in Computer Engineering** at Pimpri Chinchwad College of Engineering (Direct Second Year). I build end-to-end, production-grade systems — from intelligent mobile applications and distributed backend pipelines to blockchain-integrated platforms and AI-powered developer tooling.

With a strong foundation in **Flutter, React, FastAPI, and AI/ML**, I specialise in bridging the gap between complex engineering systems and intuitive user-facing products. My work spans mobile ecosystems, cloud-native APIs, graph databases, async task processing, and machine learning pipelines — all engineered with a focus on performance, reliability, and real-world impact.

My flagship product, **ClickNGo**, achieved 1.5K+ downloads and processed 7K+ entries in a live institutional environment — a testament to my ability to ship and scale production systems independently. I have also engineered blockchain-based streaming platforms, AI-driven codebase visualizers, and ML-powered transit optimization systems.

I approach every system with a **product engineering mindset**: understanding the user problem, designing for scale from day one, and delivering solutions that outlast the sprint cycle.

## ◈ &nbsp;Featured Projects

<br/>

<details>
<summary><b>ClickNGo &nbsp;—&nbsp; Smart Campus Visitor Management System</b></summary>

<br/>

A full-stack mobile application deployed in production at Government Polytechnic Nashik. Achieved **1,500+ downloads** and processed over **7,000 visitor entries** with QR-based registration and real-time faculty tracking — engineered, shipped, and maintained independently.

<br/>

| Attribute | Details |
|:---|:---|
| **Stack** | Flutter · Firebase · QR Code Engine · Real-Time DB |
| **Scale** | 1,500+ active installs · 7,000+ verified entries |
| **Performance** | Real-time QR registration with sub-second faculty lookup |
| **Security** | Firebase Auth · Role-based access for admin and faculty |
| **Impact** | Replaced manual paper registers at a government institution |
| **Repository** | [Play Store Deployment](https://play.google.com/store) |

<br/>

ClickNGo solves the inefficiency of manual visitor logging in institutional environments. Built entirely with **Flutter and Firebase**, the system provides QR-code-based visitor onboarding, real-time faculty availability tracking, and an admin dashboard for entry audit trails. The app was awarded **State Level Project Competition Winner** at K.K. Wagh Polytechnic, Nashik, recognizing its design, development, and execution excellence.

<br/>

</details>

<details>
<summary><b>TapToStream &nbsp;—&nbsp; Blockchain Pay-Per-View Streaming Platform</b></summary>

<br/>

A decentralized pay-per-view content streaming platform built on the Algorand blockchain. Enables secure, wallet-authenticated content access with on-chain USDC payment splitting via smart contracts — zero intermediaries, full transparency.

<br/>

| Attribute | Details |
|:---|:---|
| **Stack** | React · Node.js · Algorand · Smart Contracts · USDC |
| **Scale** | Multi-user concurrent streaming with on-chain transaction validation |
| **Performance** | Near-instant payment finality on Algorand's 4-second block time |
| **Security** | Wallet-based authentication · Smart contract payment validation |
| **Impact** | Demonstrates a viable decentralized creator monetization model |
| **Repository** | [github/taptostream](https://github.com/shreyash-devs) |

<br/>

TapToStream reimagines content monetization by removing centralized payment processors. Built with **React and Node.js** on the frontend and backend, and **Algorand smart contracts** for on-chain logic, the platform enables creators to gate content behind micro-payments in USDC. Payment splitting is automated and transparent, with wallet-based identity verification replacing traditional login systems. This project demonstrates deep understanding of decentralized finance primitives and Web3 architecture.

<br/>

</details>

<details>
<summary><b>CodeMap &nbsp;—&nbsp; AI Codebase Visualization Platform</b></summary>

<br/>

An AI-powered developer tool that parses, analyzes, and renders complex software repositories as interactive 3D knowledge graphs — accelerating codebase onboarding and architectural comprehension for engineering teams.

<br/>

| Attribute | Details |
|:---|:---|
| **Stack** | React · Three.js · FastAPI · Celery · Neo4j · Redis |
| **Scale** | Capable of mapping large multi-module repositories asynchronously |
| **Performance** | Async task processing via Celery · Real-time job tracking via Redis |
| **Security** | Isolated async workers · Repository sandboxing |
| **Impact** | Reduces codebase ramp-up time for new engineers significantly |
| **Repository** | [github/codemap](https://github.com/shreyash-devs) |

<br/>

CodeMap addresses one of software engineering's core onboarding bottlenecks: understanding unfamiliar codebases quickly. By combining **FastAPI** for RESTful analysis endpoints, **Neo4j** for graph-based dependency mapping, **Celery** for asynchronous repository processing, and **Three.js** for immersive 3D visualization, CodeMap transforms static code into explorable knowledge graphs. Engineers can traverse module relationships, identify architectural hotspots, and understand call hierarchies in minutes rather than days.

<br/>

</details>

<br/>

An ML-driven fleet management and route optimization system that predicts real-time route demand using contextual signals and allocates vehicles dynamically — modeled after the operational complexity of large-scale urban transit networks.

<br/>

| Attribute | Details |
|:---|:---|
| **Stack** | FastAPI · React · XGBoost · Google Colab · Python |
| **Scale** | Multi-route demand forecasting with dashboard visualization |
| **Performance** | XGBoost model trained on contextual features with high prediction accuracy |
| **Security** | API-gated prediction endpoints with structured request validation |
| **Impact** | Demonstrates ML-driven resource optimization for transit authorities |
| **Repository** | [github/coruscant](https://github.com/shreyash-devs) |

<br/>

Coruscant Transit Command applies supervised machine learning to the domain of urban fleet management. The **XGBoost** demand prediction model was trained on Google Colab with real-world contextual features including time-of-day, route popularity, and event signals. Predictions are served via **FastAPI** endpoints and visualized in a React-powered operations dashboard. The system demonstrates how ML can replace reactive fleet dispatch with proactive, data-driven resource allocation.
